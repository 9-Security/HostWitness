# HostWitness 開發者說明

本文件說明專案架構、如何新增 View/ViewModel 與分頁、以及如何註冊 Provider。供維護與擴充時參考。

---

## 1. 架構概觀

- **規格與分層**：見 **ARCHITECTURE.md**。整體為 Collector → Parser → Analyzer → Output；UI 使用 WPF，狀態集中於 ViewModel，MainWindow 負責視圖切換與工具列。
- **方案結構**：
  - **WinDFIR.Core**：實體（Entities）、索引（Index）、Snapshot 匯出/匯入（含 `SnapshotImporter` 對 `timeline.json` / session 事件列還原 **ProcessKey** / **NetworkFlowKey**）、設定（Settings，含 `LayoutPersistence`：版面路徑優先處理序 **`APPDATA`**）。
  - **WinDFIR.Providers**：資料採集（IProvider、EventProduced）；**`ProviderLifecycleHelper`** 與 **`ProviderStartException`**（`StartProvidersAsync` / `StopProvidersAsync`，啟動失敗時 rollback）；Registry、Offline Hive、EventLog、ETW、Browser（`BrowserHistoryProvider` 列舉設定檔目錄遇拒絕存取時記警告並略過）等。
  - **WinDFIR.UI**：WPF 主程式；Views、ViewModels、MainWindow；DataContext 為 MainViewModel。採集啟停請走 **`ProviderLifecycleHelper`**（與 Agent 共用）。
  - **WinDFIR.Agent**：主控台採集程式，無 UI；啟動/停止採集器邏輯與 UI 共用 **Providers** 內 lifecycle helper；結束代碼 **1** ＝啟動失敗（rollback）或 Snapshot 匯出失敗、**2** ＝停止失敗（不匯出；stderr 標示 export skipped）。
  - **WinDFIR.Tests**：單元測試。
- **建置與發布**：官方一鍵流程為專案根目錄 **`publish.cmd`**（由 cmd.exe 呼叫 `dotnet`）：restore → build → test → publish，預設產出 **`Release\HostWitness.exe`**。`publish.cmd` 會把 NuGet 使用者狀態隔離到 repo-local **`.nuget-appdata\NuGet`**。僅建置+測試可用 `cmd.exe /d /c .\publish.cmd -SkipPublish`。
- **分頁與內容**：上方為**動態分頁**（Timeline、Live Stream、Live Process、Live TCP），下方為**靜態分頁**（System Info、Process View、Recent Files、Prefetch、Amcache、Autorun、Event Log、Netstat、Browsing History、MFT）。目前選中的分頁由 `MainViewModel.SelectedDynamicTabIndex` / `SelectedStaticTabIndex` 與 `CurrentContentKey` 決定，工具列顯示由 `ToolbarViewType` 綁定驅動。

## 2. 新增 View / ViewModel 與分頁的步驟

若要新增一個**靜態分頁**（例如新 artifact 檢視），依下列步驟進行（動態分頁類似，改為動態 TabControl 與 DynamicTabKeys）。

### 2.1 新增 View 與 ViewModel

1. 在 `WinDFIR.UI\Views\` 新增 `YourView.xaml` 與 `YourView.xaml.cs`（UserControl）。
2. 在 `WinDFIR.UI\ViewModels\` 新增 `YourViewModel.cs`（必要時實作 INotifyPropertyChanged）；在 View 的建構函式中建立 ViewModel 並設 `DataContext = _viewModel`。
3. 若需使用 Index 或其它服務，由建構函式注入（例如 `new YourView(_index)`）。

### 2.2 在 MainWindow 註冊分頁與視圖

1. **MainWindow.xaml**  
   - 在對應的 TabControl 內新增 `<TabItem Header="顯示名稱" Name="YourTab" Tag="YourKey" ContextMenu="{StaticResource StaticTabContextMenu}"/>`。  
   - `Tag` 必須與 MainViewModel 的 key 一致（見下）。

2. **MainViewModel.cs**  
   - 在 `StaticTabKeys`（或 `DynamicTabKeys`）陣列中**依 Tab 順序**加入對應 key，例如 `"YourKey"`。  
   - 順序必須與 XAML 中 TabItem 的順序一致。

3. **MainWindow.xaml.cs**  
   - 在建構函式中：`var yourView = new YourView(...);` 並呼叫 `_viewRegistry.RegisterStaticView("YourKey", yourView)`（動態分頁用 `RegisterDynamicView`）。視圖與 Detach 狀態由 **ViewRegistryService**（`_viewRegistry`）統一管理，無需私有欄位。  
   - `GetStaticViewByKey` / `GetDynamicViewByKey` 已改為委派給 `_viewRegistry.GetStaticView(key)` / `GetDynamicView(key)`；新增 key 時只需在 MainViewModel 的 key 陣列與 XAML TabItem Tag 加入 `"YourKey"`。  
   - 若有 Detach 支援：在 `GetStaticTabByKey`（或 `GetDynamicTabByKey`）的 switch 中加入 `"YourKey" => YourTab`；Detach/Restore 邏輯會依 Tag 與 ViewRegistryService 自動處理。

### 2.3 工具列按鈕（可選）

若此分頁需要專用工具列按鈕（例如 Refresh）：

- 在 `MainWindow.xaml` 的 ToolBar 中新增按鈕，並設定  
  `Visibility="{Binding ToolbarViewType, Converter={StaticResource ToolbarViewTypeToVisibilityConverter}, ConverterParameter=YourKey}"`。  
- `ToolbarViewType` 由 `CurrentContentKey` 推導；若 key 為 `StaticProcess` 或 `Network` 會對應為同名，其餘靜態分頁為 `"Static"`（僅顯示 Detach）。若需新分頁專用按鈕，需在 `MainViewModel.GetToolbarViewType` 中為 `YourKey` 回傳專用類型（例如 `"YourKey"`），並在按鈕的 ConverterParameter 使用該類型。

### 2.4 注意事項

- Tab 的 **Tag**、**MainViewModel 的 key 陣列**、**Get*TabByKey** 必須一致且順序對應；視圖由 **ViewRegistryService** 以 key 註冊與取得。  
- 新增 key 時，若希望工具列只顯示 Detach，無需改 `GetToolbarViewType`（靜態預設即為 `"Static"`）。  
- 參考既有分頁（例如 MftView、AutorunView）的建構與 `_viewRegistry.Register*View` 呼叫方式。
- 補充：`MftView` 目前採「父 ViewModel 管理來源 tabs + `MftTabViewModel` 管理各 tab 狀態」模式；此頁沒有專用工具列 Refresh，若要重讀來源需重新執行載入。

## 3. 註冊 Provider（採集器）

1. 在 **MainWindow.xaml.cs** 建構函式中，建立 Provider 實例並加入 `_providers` 清單。  
2. 訂閱事件：`provider.EventProduced += OnEventProduced;`，使事件寫入 `_index`。  
3. 啟動時會呼叫 `ProviderLifecycleHelper.StartProvidersAsync(_providers)`；若拋出 **`ProviderStartException`**，MainWindow 會顯示錯誤並還原已啟動的採集器。關閉時停止採集器亦透過 **`ProviderLifecycleHelper.StopProvidersAsync`**。  
4. 若 Provider 需依設定啟用/停用（例如僅離線時不加入 RegistrySearchProvider），在建構時讀取 `HostWitnessSettings.Load()` 並依條件決定是否 `_providers.Add(provider)`。

參考：`RegistrySearchProvider`、`OfflineHiveRegistryProvider`、`EventLogProvider` 等之建立與加入方式。
