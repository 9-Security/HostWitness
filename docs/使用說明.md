# HostWitness 使用說明

本文說明 HostWitness 主程式（圖形介面）的安裝、介面與常用操作，供分析人員與維運人員快速上手。

---

## 1. 概述

**HostWitness** 為 Windows 單機即時取證與活動關聯工具，用於：

- 在運行中的 Windows 主機上採集程序、網路、事件紀錄、登錄檔、Prefetch、Amcache、瀏覽紀錄等證據
- 以時間軸（Timeline）關聯檔案／登錄／網路等活動
- 支援離線登錄檔（Offline Hive）、VSS 快照與 Snapshot 匯出，供後續分析或遠端回傳

**適用對象**：資安／鑑識分析人員、維運人員；需在目標主機上執行程式（本機或遠端桌面）。

---

## 2. 系統需求與執行

### 2.1 系統需求

- **作業系統**：Windows 10 / 11 或相容之 Windows Server（.NET 8.0 Windows 執行階段）
- **權限**：一般權限可執行多數功能；以下功能建議**以系統管理員身分執行**：
  - Volume Shadow Copy（VSS）用於離線 Hive、Snapshot 匯出
  - 完整事件紀錄存取、部分受保護登錄區
  - Live Process 的 Create Dump（minidump / full dump）

### 2.2 取得程式

- **建置**：`dotnet build WinDFIR.sln --configuration Release`  
  主程式輸出：`WinDFIR.UI\bin\Release\net8.0-windows\HostWitness.dll`（或經由 **`publish.cmd`** 產生 `Release\HostWitness.exe`）
- **發布**：執行專案根目錄的 **`publish.cmd`** 可產生 **`Release\HostWitness.exe` 單一檔案**（預設 self-contained **win-x64**，目標機不必安裝 .NET）；Windows on ARM 可用 `-Runtime win-arm64`；若需較小體積且目標機已裝 .NET 8，可用 `-FrameworkDependent`。

### 2.3 執行方式

- 雙擊 `HostWitness.exe` 或從命令列執行
- 首次執行會建立設定檔：`%AppData%\HostWitness\settings.json`

## 3. 主介面說明

### 3.1 整體布局

- **選單列**：File、Advanced、Help  
  - **File**：Open Snapshot…、Close Snapshot、Export Snapshot、**Export to SQLite…**、**Open from SQLite…**、**Save session now**、**Save layout now**、**Restore default layout**、Exit  
  - **Advanced**：Drill Down、Registry Search…、Settings（含「僅離線登錄檔」選項）  
  - **Help**：Export diagnostic info…、About  
- **上方分頁（動態）**：Timeline View、Live Stream、Live Process、Live TCP View  
- **下方分頁（靜態）**：System Info、Process View、Recent Files、Prefetch、Amcache、Autorun、Event Log、Netstat、Browsing History、MFT  
- **共用內容區**：顯示目前選中分頁的視圖  
- **工具列**：依分頁顯示不同按鈕（播放/暫停、重新整理、Detach、解析等）  
- **狀態列**：PID 快取與 ETW 節流狀態等（可在 Advanced → Settings 關閉）

啟動時預設選中 **System Info** 分頁。

### 3.2 分頁 Detach / Restore（分離視窗）

- **動態分頁**（Timeline、Live Stream、Live Process、Live TCP）：在分頁上**右鍵**可選「Detach to Window」將該視圖移到獨立視窗；獨立視窗工具列有「Restore to Main Window」可還原。
- **靜態分頁**（System Info、Process View、Recent Files、Prefetch、Amcache、Autorun、Event Log、Netstat、Browsing History、MFT）：同樣支援右鍵「Detach to Window」與「Restore to Main Window」。
- 主視窗工具列上的 **Detach** 圖示會依目前選中的分頁切換為「Detach」或「Restore」，點擊即可分離或還原目前分頁。

## 4. 分頁功能簡述

### 4.1 動態分頁（即時採集）

| 分頁 | 說明 |
|------|------|
| **Timeline View** | 依時間排序的活動時間軸，整合檔案、登錄、網路等事件。篩選：時間範圍（Start/End）、搜尋（全文）、Category、Action、Process、Path；**時間快捷**：Today、Last 24h、Last 7 days；**Export CSV…/Export JSON…** 匯出目前篩選結果（建議檔名含日期範圍；0 筆時會提示；對話框標題顯示將匯出筆數）；篩選結果 0 筆時顯示「目前篩選結果為 0 筆」；支援依 Process 的 Drill-down 篩選。 |
| **Live Stream** | 即時事件串流，可暫停／播放、清除。 |
| **Live Process** | 即時程序列表；可右鍵 Create Dump（minidump/full dump）、Show process tree、**Show related events (Drill down)** 跳到 Timeline 並只顯示該程序相關事件。 |
| **Live TCP View** | 即時 TCP/UDP 連線與監聽埠；可切換 TCP·UDP、解析遠端位址等。 |

### 4.2 靜態分頁（ artifact 檢視）

| 分頁 | 說明 |
|------|------|
| **System Info** | 主機名稱、OS、時區等基本系統資訊。 |
| **Process View** | 程序檢視（靜態快照），可重新整理。 |
| **Recent Files** | 近期檔案（如 Lnk、Jump List 等）。 |
| **Prefetch** | Prefetch 檔案列表與解析；詳見 `PrefetchView說明.md`。 |
| **Amcache** | Amcache 資訊；詳見 `AmcacheView說明.md`。 |
| **Autorun** | 開機／登入 Run、RunOnce、User Run、Winlogon、IFEO 等登錄與啟動項；可依 Location 篩選與搜尋。 |
| **Event Log** | 事件紀錄檢視與篩選（除傳統日誌外，於本機可用時會嘗試附加 IR Operational channel；無權限或不存在時略過）。詳見 EventLogView說明.md。 |
| **Netstat** | 網路連線／監聽埠靜態檢視，可重新整理。 |
| **Browsing History** | 瀏覽紀錄（依支援的瀏覽器）；採集時若部分使用者設定檔目錄無權限讀取，會記錄警告並略過該路徑，其餘設定檔仍會掃描。 |
| **MFT** | MFT 檢視：載入 $MFT 或 MFT 匯出檔後顯示記錄、檔名、完整路徑、建立/修改時間、In use／Deleted、Time-stomp 嫌疑等。可用 **Load from Volumes...** 一次載入多個磁區，每個來源各自開成獨立 tab。**篩選**：Record、File name、Path（子字串）、Status（All/In use/Deleted）、Time-stomp（All/Suspect only）；匯出為 CSV/JSON 時以**目前選中 tab 的篩選結果**為準。 |

## 5. 常用操作

### 5.1 開啟 Snapshot（File → Open Snapshot…）與 SQLite

- **Open Snapshot…**：選擇已匯出的 Snapshot 資料夾內的 **timeline.json**（或該資料夾），程式會載入事件並在 **Timeline View** 顯示離線時間軸；若 Snapshot 內含 `raw/` 證據檔，evidence reference 會自動解析到目前這個 Snapshot 的本地路徑。載入後選單會出現 **Close Snapshot**，點擊可還原為即時採集時間軸。
- **多個 `snapshot_*` 子目錄**：若開啟的是「父資料夾」且其下有多個 `snapshot_*`，程式會依路徑**不區分大小寫排序**後，載入**第一個**含有 `timeline.json` 的 bundle（與完整性驗證腳本選取規則一致）。建議分析時直接選定目標 `snapshot_yyyyMMdd_HHmmss` 資料夾以避免歧義。
- **事件中的程序／網路身分**：離線載入時會正確還原 `subjectProcess`（ProcessKey）與 `objectNetworkFlow`（NetworkFlowKey），以利 Timeline 鑽取與網路關聯與匯出前一致。
- **Export to SQLite…**：將目前時間軸（或已開啟的 Snapshot）匯出為一個 .db 檔，供離線查詢或攜帶。
- **Open from SQLite…**：選擇先前匯出的 .db 檔，會以分頁方式載入時間軸並以 Snapshot 方式顯示（大型 DB 較不易卡住）。

### 5.2 匯出 Snapshot（File → Export Snapshot）

- 將目前 Activity Index 內的事件與相關資料匯出為 Snapshot 資料夾（內含 timeline、entities、manifest、raw 等）；可攜式 raw 證據檔會複製到 `raw/`，並把 `timeline.json` 中對應的 evidence reference 改寫成 bundle 內相對路徑。
- `manifest.json` records `modeProfile`, `registryMode` / `registryLiveEnabled`, `preflight` (`executionContext`, `isAdministrator`, `vssServiceRunning`, `useVssSnapshots`, `timeZoneDisplay`, `enabledProviders`, `outputDirectory`, `outputDirectoryWritable`, `availableFreeSpaceBytes`, `warnings`, `errors`; Agent may also include `collectSeconds`), `collectionSummary` (`sourceEventCount`, `exportedEventCount`, `eventCap`, `wasEventCountCapped`, `evidenceReferenceCount`, `rewrittenEvidenceReferenceCount`, `copiedArtifactFileCount`, `skippedEvidenceReferenceCount`, `failedEvidenceReferenceCount`, `wasArtifactCopyIncomplete`, `usedVssSnapshotForArtifactCopy`, `artifactCopyWarningCount`, `preflightWarningCount`, `preflightErrorCount`, `etwDroppedEventTotal`, `uiBackpressureDroppedTotal`), `knownLimitations`, `etwTotalDrops` (when throttling occurred), and `uiBackpressureDroppedTotal` for UI exports.
- 若啟用 VSS，會盡量從陰影複本讀取鎖定中的 Hive；需管理員與 Volume Shadow Copy 服務。
- 匯出路徑可自選；結果為 `snapshot_yyyyMMdd_HHmmss` 子目錄。

### 5.2a Session 還原與手動儲存

- **關閉時自動儲存**：程式關閉時，若目前時間軸**至少有一筆事件**，會將內容寫入 `%AppData%\HostWitness\last_session`；若**沒有任何事件**，會**清除**已儲存的 session 檔（`timeline.json` / `meta.json`），避免下次啟動還原到過期的舊案件。
- **啟動時還原**：下次啟動時若存在上次 session，會詢問是否還原；若目前 **Activity Index Max events** 小於 session 筆數，對話框會先提示只保留最新可容納筆數。選「是」後程式會載入事件並刷新 Timeline。
- **File → Save session now**：可隨時手動儲存目前時間軸為「上次 session」，供下次啟動還原使用。

### 5.2b 版面儲存與還原

- **關閉時自動儲存**：程式關閉時會將視窗位置、大小與目前分頁選中狀態寫入 `%AppData%\HostWitness\layout.json`，下次啟動時自動套用。實作上若處理序環境變數 **`APPDATA`** 已設定則**優先**以其為基底路徑（下層仍為 `HostWitness\layout.json`），與一般 Windows 工作階段一致，並便於測試或特殊啟動情境。
- **File → Save layout now**：立即儲存目前版面（含 detached 視窗位置/大小），下次啟動時還原。
- **File → Restore default layout**：清除儲存的版面並還原為預設（System Info 分頁、視窗置中）。

### 5.3 Drill Down（依程序篩選時間軸）

- 在 **Live Process** 分頁選取一筆程序後：  
  - **雙擊**該列，或  
  - 右鍵 **「Show related events (Drill down)」**，或  
  - 選單 **Advanced → Drill Down**  
  會切換到 **Timeline View**，並只顯示該 Process（PID）相關的檔案／登錄／網路等事件。
- 在 Timeline 上可看到「Filtered by PID: xxx」與 **Clear process filter**，點擊即可還原為全量時間軸。

### 5.4 設定（Advanced → Settings）

- **PID 快取**：EventLog / ETW 使用的 TTL、LRU、長駐清單等。
- **字型大小**：全介面字型（Small / Medium / Large / Custom）。
- **TimeZoneDisplay**：Timeline 等處時間顯示為「本機時間」或「UTC」。
- **狀態列診斷（Status Bar Diagnostics）**：是否顯示狀態列診斷資訊（PID 快取 / ETW 節流 / UI Queue 背壓）。需重啟套用。
- **Registry（鑑識）**：預設為「僅離線登錄檔」。若要使用 Live Registry，需先取消勾選 offline-only，再勾選「允許 Live Registry（實驗性，non-forensic）」並重啟。未啟用時，Registry Search 會顯示 policy disabled 提示並停用 Live 查詢。
- **Profiles**：Settings 視窗可一鍵套用 `Forensic Strict`（離線/保守）或 `Triage Fast`（即時/高吞吐）組態；也可 `Export Profile...` / `Import Profile...` 匯出匯入 JSON。
- **Raw Disk (Offline Hive)**：可從實體磁碟讀取 Hive（需管理員）。每行一筆：`DriveNumber,OffsetBytes,SizeBytes,HiveName`（例：`0,0,409600,SYSTEM`）。重啟後套用。
- **Export**：可設定「Export default path」作為匯出對話框的預設目錄（Timeline CSV/JSON 等）。
- **Activity Index Max events**：時間軸可保留的最大事件數（0 表示不限制）；超過時會依策略修剪舊事件。輸入 0 時會顯示 OOM 風險提示。
- **ETW Cache**：除 TTL/MaxEntries 外，可設定 **Throttle cap (events/s)**（0 = 內建預設）；提高可減少丟棄但可能增加 CPU/記憶體。

設定會寫入 `%AppData%\HostWitness\settings.json`。

### 5.5 MFT 檢視與匯出

- **為何 MFT 是空的？** MFT 分頁不會自動載入資料，需手動選擇來源。
- 在 **MFT** 分頁點「Load MFT file...」可載入單一 `$MFT` / MFT 匯出檔；程式會自動偵測常見 MFT record size（1024 / 4096 bytes），並把採用的大小寫到該 tab 狀態列。或點「**Load from Volumes...**」一次選擇一個以上磁碟機（如 `C:`、`D:`）。
- 使用 **Load from Volumes...** 時，程式會為每個來源建立各自的 tab；磁碟機來源會依序嘗試 **raw volume**、**Backup privilege 直接讀取 `$MFT`**、**VSS snapshot**。需系統管理員。
- **無法載入時**：請確認該磁碟機是**本機、已解鎖、NTFS** 分割區；若仍失敗，改以「Load MFT file...」載入事先匯出的 `$MFT` 檔。各 tab 的狀態列會顯示 raw volume / Backup privilege / VSS 的失敗原因。
- **讀取上限提示**：磁碟型來源若命中 100 MB 讀取上限，狀態列會明確提示僅載入前段資料，結果可能不完整。
- **不提供 Refresh**：為避免已載入結果被清空或重跑，目前若要重讀某個來源，請再次使用「Load MFT file...」或「Load from Volumes...」；相同來源會覆寫原本 tab。
- **每個 tab 都有自己的分頁與篩選**：單頁預設顯示 500 筆；可於「每頁筆數」選擇 100／250／500／1000／2000／5000；「第一頁／上一頁／下一頁／最後一頁」切頁，或於「Jump to [輸入]」輸入頁碼後按 Go。
- **Export CSV... / Export JSON...**：將目前**選中 tab 的篩選後全部**清單匯出，不受該 tab 當前頁碼影響。
- 已刪除記錄會以灰底顯示；1980 年前的 NTFS 時間戳不會再被直接清空；Time-stomp? 為 True 表示 `$STANDARD_INFORMATION` 與 `$FILE_NAME` 時間差異較大，可能為時間竄改嫌疑。

### 5.6 匯出診斷資訊（Help → Export diagnostic info…）

- 將版本、機器名稱、設定檔路徑、Log 資料夾、Index 事件數（與是否載入 Snapshot）及 startup.log 最後幾行匯出為文字檔，方便除錯或回報問題。

### 5.7 鍵盤快速鍵

| 快速鍵 | 功能 |
|--------|------|
| **Ctrl+O** | 開啟 Snapshot（File → Open Snapshot…） |
| **Ctrl+W** | 關閉 Snapshot（File → Close Snapshot） |
| **Ctrl+E** | 匯出 Snapshot（File → Export Snapshot） |
| **Ctrl+L** | 清除 Timeline（僅在 Timeline 分頁時；清除時間範圍、篩選與 process filter） |
| **Ctrl+Shift+D** | Drill Down（Advanced → Drill Down；從 Live Process 跳至 Timeline 並篩選該程序） |

## 6. 設定檔與進階

- **設定檔路徑**：`%AppData%\HostWitness\settings.json`
- **Ui.ExportDefaultDirectory**：匯出對話框預設目錄（可選）。
- **Ui.ShowStatusBarDiagnostics**：是否顯示狀態列診斷資訊（PID 快取 / ETW 節流 / UI Queue 背壓）；需重啟套用。
- **Ui.RegistryUseOfflineOnly**：為 true 時僅使用離線登錄檔，不使用 Live Registry Provider。
- **Ui.EnableLiveRegistryExperimental**：為 true 且 RegistryUseOfflineOnly=false 時，才允許 Live Registry（non-forensic）。
- **Ui.RawHiveSources**：Raw Disk 離線 Hive 清單（見 Settings → Raw Disk）；重啟後套用。
- **Index.MaxEvents**：Activity Index 最大事件數（0 = 無上限）。
- **版面**：`%AppData%\HostWitness\layout.json` 儲存視窗位置、大小與分頁選中；關閉時自動寫入。
- **VSS**：離線 Hive 與 Snapshot 匯出會嘗試使用 Volume Shadow Copy；失敗時自動回退即時路徑。詳見 `LIMITATIONS.md`。
- **ETW 節流**：高頻 ETW 事件可能被節流，部分事件會被丟棄；狀態列會顯示節流相關資訊。
- **UI Queue 背壓**：狀態列顯示 UI queue pending/dropped。當事件生產速率高於 UI 渲染速率時，系統會丟棄部分 UI 佇列事件以維持介面回應。

## 7. 遠端採集（Agent）

若需在遠端主機上無 UI 採集並回傳結果，請使用 **HostWitness.Agent**：

- 主控台程式，執行一段時間後將結果匯出為 Snapshot 資料夾。
- 支援參數：`[輸出目錄] [採集秒數] [--etw] [--providers=名稱1,名稱2,...]`（**ETW 非預設**：需加 `--etw`，或於 `--providers=` 清單內含 `ETW`。）
- **結束代碼**：`0` = 成功匯出；`1` = 採集器啟動失敗（rollback）或 Snapshot **匯出** 失敗；`2` = 採集結束後停止採集器失敗（**不匯出**，stderr 標示 export skipped）。排程與回傳腳本應檢查 exit code。
- 專案內 **scripts/ScheduledCollect.ps1**、**scripts/RunCollectAndCopy.ps1** 提供排程與自動回傳範例。
- 回傳腳本會在輸出目錄寫入 `collect-status-latest.json`（含 success/stage/error、可選 agentExitCode／exportSkipped、snapshotPath、destinationPath、manifestVerified）；可加 `-StatusHmacKey` 產生 HMAC-SHA256 簽章。
- 可用 `scripts/VerifyCollectStatusSignature.ps1 -StatusPath <collect-status-latest.json> -HmacKey <shared-secret>` 驗證狀態檔簽章。
- 可用 `scripts/VerifySnapshotIntegrity.ps1 -SnapshotPath <snapshot_*>` 單筆驗證，或 `-SnapshotRoot <path> -Recurse` 批次驗證（比對 `hashes.txt`）。
- 可用 `scripts/VerifyCollectionPipeline.ps1 -StatusPath <collect-status-latest.json> -SnapshotPath <snapshot_*> [-HmacKey ...]` 一鍵驗簽 + 驗完整性，輸出 `pipeline-verify-report.json`。
- 詳見 **遠端採集Agent說明.md**。

## 8. 限制與參考文件

- **離線 Hive 增強欄位**：SOFTWARE/SYSTEM 下可降低調讀 **BITS**、**WMI**（CIMOM 與 ControlSet*\\Control\\WMI\\Security）、**SRUM**（登録子樹；不含 SRUDB.dat 解析）等保守結構化欄位；詳見 LIMITATIONS.md §16。
- **取證與即時採集限制**：見 **LIMITATIONS.md**（即時系統一致性、權限、ETW 節流、不進行修復或殺程序等）。
- **撰寫鑑識報告時**：應引用匯出 Snapshot 內之 **manifest.json**（含 `modeProfile`、`preflight`、`collectionSummary`、`knownLimitations`、`registryMode`、`etwTotalDrops` 等欄位），並於報告中註明工具限制與假設，參考 **LIMITATIONS.md**。必要時可一併提供 Help → Export diagnostic info 產出之診斷檔。
- **架構與假設**：**ARCHITECTURE.md**、**FORENSIC_ASSUMPTIONS.md**。
