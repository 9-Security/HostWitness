# 建置與發布（維護者）

本檔給維護者參考；一般使用者請看根目錄 [README.zh-TW.md](../README.zh-TW.md)。

## 編譯與執行

- **發布腳本**：在專案目錄執行 **`publish.cmd`**（由 **cmd.exe** 呼叫 `dotnet`，官方唯一支援的一鍵發布路徑）：`dotnet restore` → build → **單元測試** → **publish**；預設產出 **`Release\HostWitness.exe` 單一檔案**（**self-contained** + **win-x64**）。`publish.ps1` 已停用，請改用 `publish.cmd`。
- **啟動**：`啟動HostWitness.bat`（一般權限）或 `啟動HostWitness(管理員).bat`（管理員權限），皆從 `Release\HostWitness.exe` 啟動。
- **Stable scope note**：目前 stable 更新聚焦於本機 UI/Core 流程；`HostWitness.Agent` 保留於 repo 作為後續更新項目，不在目前 release gate 範圍內。

```bat
REM 預設：self-contained 單檔 win-x64 -> Release\HostWitness.exe
cmd.exe /d /c .\publish.cmd

REM Windows on ARM64
cmd.exe /d /c .\publish.cmd -Runtime win-arm64

REM 較小輸出；目標機需 .NET 8 Desktop Runtime
cmd.exe /d /c .\publish.cmd -FrameworkDependent

REM 僅建置 + 測試（不產生 Release\ 輸出）
cmd.exe /d /c .\publish.cmd -SkipPublish

REM 先跑 stable gate 再 publish
cmd.exe /d /c .\publish.cmd -StableGate
```

- **publish.cmd troubleshooting**：`InvokeStableReleaseGate.ps1` 透過 cmd.exe 執行 **`publish.cmd -SkipPublish`**。`publish.cmd` 與 stable gate 會把 NuGet 狀態隔離在 **`.nuget-appdata\NuGet`**，避免使用者 `%AppData%\NuGet` 設定損毀影響官方發布路徑。若 restore/build/test 仍失敗，先看 **publish.cmd: ERROR** 的步驟名稱；檔案鎖定失敗通常代表 `HostWitness.exe`、`Release\` 或 `bin\Release` 正被使用。若本機 wrapper 快取看似損毀，移除 `.nuget-appdata\` 後重跑 `cmd.exe /d /c .\publish.cmd`。

手動等同預設發布：

```
dotnet publish WinDFIR.UI\WinDFIR.UI.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeAllContentForSelfExtract=true -p:DebugType=none -o Release
```

## 程式碼簽章

> 公開信任（public trust）的目標是改用 **Certum 開源程式碼簽章（雲端 SimplySign）**。在取得該憑證前，repo 內提供**自簽憑證**流程，僅供本機開發/測試使用——自簽不會被外部信任，下載者仍會看到 SmartScreen 警告。

自簽憑證（本機開發用）：

- 首次：執行 `.\create-signing-cert.ps1` 建立自簽程式碼簽章憑證，匯出至 `certs\HostWitness.pfx`。Subject 使用純 ASCII（L=Taipei、S=Taiwan、C=TW）以避免簽章詳細資訊出現亂碼。憑證與 PFX 勿提交版控（`.gitignore` 已含 `certs/`、`*.pfx`）。
- 發布時：若存在 `certs\HostWitness.pfx`，`publish.cmd` 會呼叫 `scripts\Sign-HostWitness.ps1` 以 signtool 簽章 `Release\HostWitness.exe`（/d、/du、**/tr** 時間戳記）。可用環境變數覆寫：`HOSTWITNESS_PFX_PASSWORD`（PFX 密碼）、`HOSTWITNESS_SIGNING_URL`（描述 URL）、`HOSTWITNESS_TIMESTAMP_URL`（時間戳記伺服器，預設 `http://timestamp.digicert.com`）。無憑證或簽章失敗時仍會完成發布，主控台會提示 exe 可能未簽章。
- 簽章容錯：`Sign-HostWitness.ps1` 依序嘗試 RFC3161 (`/tr`)、舊式時間戳 (`/t`)、無時間戳。**dotnet publish** 失敗時 `publish.cmd` 以非零結束；僅簽章步驟失敗不讓 `publish.cmd` 失敗。
- 自簽憑證驗證：`Get-AuthenticodeSignature` 可能顯示 `UnknownError`（root not trusted），代表簽章存在但本機信任鏈未建立；若要顯示為 `Valid`，需將自簽根憑證匯入受信任根憑證存放區。
- 需安裝 **Windows SDK**（含 signtool.exe）。

## 發布前檢查清單

改完 code 後建議依下列步驟再發布，以確保穩定與輸出正確：

1. **建置與測試**：執行 `cmd.exe /d /c .\publish.cmd`。腳本會依序：restore → Build → **單元測試** → **Publish**（self-contained 單檔），並以 repo-local **`.nuget-appdata\NuGet`** 隔離 NuGet 設定。若測試失敗會中止，不會產生完整 Release 輸出。
2. **手動驗證（建議）**：以**管理員**與**一般使用者**各執行一次：啟動程式 → 採集約 30 秒 → 匯出 Snapshot → 關閉程式。確認無崩潰且匯出可完成。
3. **Manifest checks**：verify that `manifest.json` includes `modeProfile`, `preflight` (`executionContext`, `isAdministrator`, `vssServiceRunning`, `timeZoneDisplay`, `enabledProviders`, `outputDirectoryWritable`, `availableFreeSpaceBytes`, `warnings`, `errors`; Agent may also include `collectSeconds`), `collectionSummary` (`sourceEventCount`, `exportedEventCount`, `eventCap`, `wasEventCountCapped`, `evidenceReferenceCount`, `rewrittenEvidenceReferenceCount`, `copiedArtifactFileCount`, `skippedEvidenceReferenceCount`, `failedEvidenceReferenceCount`, `wasArtifactCopyIncomplete`, `usedVssSnapshotForArtifactCopy`, `artifactCopyWarningCount`, `preflightWarningCount`, `preflightErrorCount`, `etwDroppedEventTotal`, `uiBackpressureDroppedTotal`), `knownLimitations`, `registryMode` / `registryLiveEnabled`, and `etwTotalDrops` (when throttling occurred).
4. **Stable gate**：run `powershell -ExecutionPolicy Bypass -File .\scripts\InvokeStableReleaseGate.ps1` before a stable publish. The gate runs **`publish.cmd -SkipPublish`** (via cmd.exe) for build+test, also under repo-local **`.nuget-appdata\NuGet`**, writes JSON reports to `Release_Verification\`, and checks required release docs. One-step gate+publish: `cmd.exe /d /c .\publish.cmd -StableGate`.

目前釋出流程以 **`publish.cmd`** 與 `Release\` 為準；`publish.ps1` 已停用。`Release_Verification\` 僅保留正式 release 與最新 workspace 驗證報告。詳細的穩定版檢查與驗證流程另見 `docs\StableReleaseChecklist.md`、`docs\StableReleaseManualVerificationRunbook.md`。
