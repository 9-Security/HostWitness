# HostWitness 穩定版邊界定義

本文件定義目前 HostWitness 作為「穩定版現場 DFIR 工具」時的支援邊界、模式邊界、輸出邊界與 release 約束。

用途：

- 讓維護者知道哪些行為可當作 stable baseline。
- 避免後續功能擴充破壞既有 DFIR 假設。
- 作為 `Forensic Strict` / `Triage Fast`、`snapshot` / `SQLite`、高風險功能的共同判準。

## 1. 穩定版目前定義

HostWitness 目前的穩定版定位是：

- Windows 單機 DFIR 工具
- 支援 live triage 與 forensic-preferred offline / snapshot workflow
- 以本機互動式 UI 為主；`HostWitness.Agent` 目前保留於 repo，但不納入本輪 stable update 完成定義
- 以「可辯護、可重現、可支援」優先於新增更多 artifact parser

穩定版不代表：

- 所有 live 資料都具備單一時間點一致性
- 所有功能都屬 forensic-safe
- 所有 Windows 版本與環境都已完成正式驗證矩陣

---

## 2. 平台與執行邊界

### 2.1 平台

- 所有專案目前都以 `net8.0-windows` 為目標框架。
- UI 是 WPF 桌面應用程式。
- Agent 是 Windows console executable。

因此目前穩定版邊界是：

- Windows only
- local host first
- 不將 Linux / macOS / cross-platform 視為支援範圍

### 2.2 權限

- 多數檢視與離線分析可在一般權限下執行。
- 以下功能應視為需要管理員或至少建議管理員：
  - VSS snapshot
  - Raw Disk / Offline Hive raw source
  - `Load from Volumes...` 的 MFT volume path
  - process dump 對其他程序的操作

### 2.3 目前可宣稱與不可宣稱

目前可宣稱：

- 以 Windows 本機為目標的 UI / Agent 採集與離線載入流程
- `snapshot` 匯出 / 開啟
- SQLite 匯出 / 開啟
- `Forensic Strict` 與 `Triage Fast` 兩組明確 profile

目前不應宣稱：

- 已完成正式 Windows 10 / 11 / Server 驗證矩陣
- 所有 live 採集結果具 point-in-time consistency
- 所有功能在非管理員環境下都等價可用

在正式驗證矩陣完成前，Windows Server 與特殊環境應視為 best-effort。

## 3. 模式邊界

## 3.1 `Forensic Strict`

這是目前應優先視為穩定版預設的 forensic-first 模式。

目前設定值：

- `RegistryUseOfflineOnly = true`
- `EnableLiveRegistryExperimental = false`
- `EtwThrottleMaxPerSecond = 300`
- `IndexMaxEvents = 200000`
- `EventLogProvisionalTtlMinutes = 720`
- `EventLogAuthoritativeTtlMinutes = 1440`
- `EventLogMaxEntries = 20000`
- `EtwProvisionalTtlMinutes = 30`
- `EtwAuthoritativeTtlMinutes = 120`
- `EtwMaxEntries = 50000`
- `LongLivedTtlMinutes = 10080`

補充：

- 新產生的預設設定與舊版 stable 預設設定，最終都應落在這組值上；不應默默以 `custom` 顯示。

定位：

- offline-first
- registry 走 forensic-safe default
- 記憶體與 ETW 節流較保守
- 適合正式採證、報告保守引用與高可辯護場景

## 3.2 `Triage Fast`

這是目前應視為 live triage 模式，而不是 forensic-safe default。

目前設定值：

- `RegistryUseOfflineOnly = false`
- `EnableLiveRegistryExperimental = true`
- `EtwThrottleMaxPerSecond = 800`
- `IndexMaxEvents = 100000`
- `EventLogProvisionalTtlMinutes = 240`
- `EventLogAuthoritativeTtlMinutes = 720`
- `EventLogMaxEntries = 12000`
- `EtwProvisionalTtlMinutes = 15`
- `EtwAuthoritativeTtlMinutes = 60`
- `EtwMaxEntries = 25000`
- `LongLivedTtlMinutes = 4320`

定位：

- triage-first
- wider live visibility
- 容許 Live Registry experimental 路徑
- 不應在文件或 UI 中被描述為 forensic-safe

## 3.3 穩定版模式規則

- 不得將 `Triage Fast` 描述成鑑識安全預設。
- 不得將 Live Registry 改成預設啟用。
- 若 profile 數值或行為有異動，必須同步更新：
  - [使用說明](D:/cursor/IR_script/HostWitness/docs/使用說明.md)
  - [穩定版路線圖](D:/cursor/IR_script/HostWitness/docs/穩定版路線圖.md)
  - 本文件

## 4. 功能分級

### 4.1 forensic-preferred baseline

以下功能可視為目前穩定版中較接近 forensic workflow 的基線：

- Offline Hive 分析
- VSS-backed Offline Hive / Snapshot 路徑
- `Load MFT file...`
- `Load from Volumes...` 的 MFT 讀取流程
- Snapshot export / open
- SQLite export / open

注意：

- `Load from Volumes...` 仍受 100 MB 讀取上限、管理員權限、本機已解鎖 NTFS 等限制。
- `Load MFT file...` 目前自動偵測常見 `1024 / 4096` record size，但非常規 layout 仍需外部驗證。
- Snapshot export 仍受 event cap 與 VSS / live fallback 限制。

### 4.2 stable live triage baseline

以下功能屬穩定版產品範圍內的 live triage 能力，但結果仍受 live system 影響：

- Timeline
- Live Stream
- Live Process
- Live TCP
- Event Log
- Netstat
- Browsing History
- Prefetch
- Amcache
- Autorun
- Recent Files

注意：

- live 視角本質上受當前主機狀態、API hooking、ETW throttling、權限與資料即時變動影響。
- 這些功能可穩定使用，不代表其輸出天然等於 forensic-safe evidence。

### 4.3 明確 non-forensic / high-risk

以下功能應在穩定版中明確標示為 non-forensic、experimental 或 high-risk：

- Live Registry collection / search
- process dump
- 任何可能改變現場狀態的主動操作

規則：

- UI 必須明確標示
- 文件必須明確標示
- 後續若納入 summary / manifest，應留下痕跡

操作員 profile、SOP 與 UI 動作分級（forensic / non-forensic / advanced）見 `docs/StableOperatorProfileAndSOP.md`。

## 5. 輸出邊界

### 5.1 Snapshot

目前 Snapshot bundle 的穩定邊界為：

- `timeline.json`
- `entities.json`
- `manifest.json`
- `hashes.txt`
- `raw/`

目前 manifest 基本欄位包括：

- `toolVersion = current executable version`（例如 `1.0.1`）
- `collectionTime`
- `host.hostname`
- `host.machineSid`
- `host.osVersion`
- `snapshotFormat = 1.0`

此外還可能包含匯出時附加欄位，例如：

- `collectionSummary`
- `modeProfile`
- `preflight`
- `knownLimitations`
- `registryMode`
- `registryLiveEnabled`
- `etwTotalDrops`
- `uiBackpressureDroppedTotal`

### 5.2 SQLite

目前 SQLite 輸出邊界為：

- `schema_version = 2`
- `events` table
- `meta` table

### 5.3 輸出變更規則

若變更以下任一項目，應視為高風險修改：

- `manifest.json` 欄位
- `snapshotFormat`
- SQLite schema / migration
- 匯出上限或截斷語意
- `collectionSummary` / `modeProfile` / `preflight` / `knownLimitations` / `registryMode` / `etwTotalDrops` 這類模式與風險欄位

高風險輸出變更必須：

- 先 review，再 patch
- 同步更新文件
- 說明 migration / backward compatibility

---

## 6. 目前穩定版不可隨意改動的規則

- 不改變 MFT volume load 的 fallback 順序：
  - `raw volume -> Backup privilege -> VSS`
- 不移除 MFT 截斷警告與 record-size detection 的使用者可見性
- 不把 Live Registry 變成預設路徑
- 不把 high-risk 功能混進預設 forensic workflow
- 不在未更新版本與 migration 規則前更動 Snapshot / SQLite 輸出語意
- 不刪除或弱化 `knownLimitations` 類型的風險揭露

---

## 7. 穩定版 release 前最低檢查

每次 stable release 前，至少應確認：

- [ ] build / test 維持綠燈
- [ ] `Forensic Strict` 與 `Triage Fast` 行為未漂移，或文件已同步
- [ ] Snapshot / SQLite 輸出邊界未被無聲破壞
- [ ] `LIMITATIONS.md`、[使用說明](D:/cursor/IR_script/HostWitness/docs/使用說明.md)、本文件同步
- [ ] non-forensic / high-risk 功能仍有清楚標示

---

## 8. 目前狀態與剩餘工作

依目前路線圖，repo 內的 stable 路線開發已完成：

1. 模式與告警落盤
2. `preflight check`
3. `collection summary`
4. stable release gate / SOP / manual verification 文件

剩餘工作不再是新功能開發，而是 release 驗證與簽核：

- 執行 manual verification matrix
- 完成 verification record 與 checklist
- 做 stable sign-off / candidate 發佈
